# 粤语填词系统 — 逐步实施计划

> **系统一句话概要**：普通话分词提取语义槽 → 简谱解析成字位与 0243/02493 模板 → 词典检索出可唱候选 → GLM 在 JSON 约束下逐句填词 → 规则评分器重排 → GLM 做有限润色。

---

## 〇、版本与模型约定

| 项目 | 决定 |
|------|------|
| 对话接口 | `https://open.bigmodel.cn/api/paas/v4/chat/completions` |
| 鉴权方式 | Bearer API Key |
| 批量候选生成模型 | `glm-5-turbo` |
| 重排/润色模型 | `glm-5` 或 `glm-5.1` |
| 模型名不写死 | 配置文件中维护 `model_name` 字段，不硬编码 |
| 输出格式 | 优先使用 `json_object` 结构化输出 + 函数调用 |

## 一、输入格式规范

系统接受一个结构化输入对象（对应 `src/input/schema.py` 中的 `LyricInput`），字段如下：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `jianpu` | 字符串 | 是 | 简谱，用 `\|` 按小节分隔 |
| `mandarin_seed` | 字符串 | 是 | 普通话语义种子，用 `\|` 按小节分隔，`\|` 数量必须与 `jianpu` 一致 |
| `theme_tags` | 字符串列表 | 否 | 主题/情绪标签，如 `["怀旧", "爱情"]` |
| `style_tags` | 字符串列表 | 否 | 风格标签，如 `["抒情", "口语化", "热血"]` |

---

### 1.1 `jianpu` 字段格式

用一行字符串表示完整简谱。**符号规则**：

| 符号 | 含义 | 示例 |
|------|------|------|
| `1`–`7` | 音高（do–ti） | `3` |
| `0` | 休止符/空拍，不占字位，但保留在走向推断中 | `0` |
| `^` 前缀 | 高八度 | `^1` |
| `,` 前缀 | 低八度 | `,5` |
| `-` 后缀或独立 token | 延音，每个 `-` 延长一拍；可紧接音符（`3-`）或空格分隔（`3 -`），解析器均归属于**前一个有音高的音符** | `3-`、`3 -`（均为两拍）、`^1 - - -`（四拍） |
| `.` 后缀 | 附点，时值延长原音符的一半 | `3.`（四分附点=1.5拍）、`3_.`（八分附点=0.75拍） |
| `_` 后缀 | 八分音符（半拍） | `3_` |
| `=` 后缀 | 十六分音符（四分之一拍） | `3=` |
| `()` | 连音/一字多音，括号内只占一个字位，音高按括号内第一个有音高的音符计算；括号可跨小节 | `(1 2 3)`、`(5 6 \| 7 ^1)` |
| `(空格)` | 分隔音符 | `1 2 3` |
| `\|` | 小节线，同时是与 `mandarin_seed` 的**小节对齐单元** | `1 2 \| 3 4` |

**附加规则**：

- 强拍/弱拍由解析器根据小节内位置和时值自动推断，不需用户标注
- 若某小节所有音符均为 `0`，视为**纯伴奏小节**，对应 `mandarin_seed` 段留空，跳过填词

---

### 1.2 `mandarin_seed` 字段格式

用 `|` 把普通话语义内容划分为与简谱等数量的小节段，每段对应一个小节的语义预算。

**规则**：

- `|` 数量必须与 `jianpu` 的 `|` 数量严格相同
- 纯伴奏小节对应的段**留空**（两侧 `|` 之间只有空白字符）
- 每段不要求字数与字位数相等，只要求**语义不跨小节**
- `|` 两侧空格可选，解析时忽略
- 普通话内容仅作**语义种子**，不做逐字翻译

---

### 1.3 校验规则

以下校验在 schema 层完成，不一致时抛出 `ValueError`，不进入后续任何处理：

```python
if jianpu.count('|') != mandarin_seed.count('|'):
    raise ValueError("jianpu 与 mandarin_seed 的小节数（| 数量）不一致")
```

---

### 1.4 完整示例

```
jianpu:         "1_ 2_ 3 5- | 0 0 0 0 | 5 3- 2 1"
mandarin_seed:  "风吹过旧街灯 |  | 往日已不再"
theme_tags:     ["怀旧"]
style_tags:     ["抒情"]
```

解析结果：

| 小节 | jianpu 内容 | 可唱字位数 | mandarin_seed 段 | 处理方式 |
|------|------------|-----------|-----------------|----------|
| 1 | `1_ 2_ 3 5-` | 4 | 风吹过旧街灯 | 正常填词 |
| 2 | `0 0 0 0` | 0 | （空） | 纯伴奏，跳过 |
| 3 | `5 3- 2 1` | 4 | 往日已不再 | 正常填词 |

## 二、里程碑规划（四阶段 MVP）

### 阶段 A — 单句端到端原型
> 输入一句普通话语义种子 + 一句简谱，输出 ≥10 个粤语候选句。

### 阶段 B — 协音过滤
> 接入本地粤语读音库与 0243/02493 映射，过滤不协音结果。

### 阶段 C — 韵脚与上下文
> 加入句尾押韵、主题一致性、上下句衔接。

### 阶段 D — 高级规则
> 引入"同向原理"、"弱拍放松"等高级规则；支持多句/整段。

- [ ] **P1-4** 生成 **走向模板**

- [ ] **P4B-1** 实现 `scorer.py` 总评分函数
  ```
  总分 = 0.45 × 协音 + 0.20 × 语义保持 + 0.15 × 口语自然度 + 0.10 × 分句匹配 + 0.10 × 韵脚/风格
  ```
  权重从 `settings.yaml` 读取，可调。

- [ ] **P4B-2** 实现协音子评分（占总分 0.45 的部分）

  - 句尾协调：尾字音高与旋律末音的配合

- [ ] **P4B-3** 实现硬模板 vs 软模板机制

  - 软模板：弱拍位置允许同向近邻替代（可配置容差等级）
  - 默认先严后松，避免模型钻规则空子

- [ ] **P4B-4** 实现语义保持评分（0.20）

- [ ] **P4B-5** 实现口语自然度评分（0.15）

- [ ] **P4B-7** 实现韵脚/风格评分（0.10）

## 五、关键设计决策记录

1. **模型名不写死**：配置文件维护，支持 `glm-5` / `glm-5-turbo` / `glm-5.1` 切换。
2. **内部精细、外部简写**：内部用九声→五类/六调，外部展示用 0243。
3. **两阶段生成**：槽位填充（结构化 JSON）+ 句级润色（有限改写），分离"协音正确"和"语言自然"。
4. **规则优先于模型**：规则验证器做扎实，模型可替换；规则层不稳，换再强模型也写出不啱音的词。
5. **先严后松**：软模板是可配置的后验容错，不是前置默认放松。
6. **普通话输入当语义种子**：不逐字翻译，而是提取意象/动作/情绪供粤语重新组句。
7. **本地词典不依赖爬虫**：构建本地 SQLite/JSON 索引，不运行时抓取外部网页。
